📖 Introduction

Qu'est-ce que Cat Classifier ?

Cat Classifier est une application de machine learning qui utilise la régression logistique pour classifier les images comme contenant un chat ou non. Ce projet démontre l'implémentation d'un réseau de neurones simple pour la classification d'images.

🎯 Objectifs pédagogiques
  • Comprendre les fondamentaux de la régression logistique
  • Implémenter un réseau de neurones à une couche
  • Créer une API web avec Flask
  • Déployer une application de machine learning
🔬 Technologies utilisées
  • Backend: Python, Flask
  • Machine Learning: NumPy, SciPy
  • Traitement d'images: PIL, Matplotlib
  • Frontend: HTML5, Bootstrap 5
  • Dataset: Cats vs Non-cats (64x64)
  • Algorithm: Logistic Regression

Modèle Machine Learning
🧠 Régression Logistique

Le modèle utilise une régression logistique vue comme un réseau de neurones à une couche :

Forward Propagation:
$z = w^T x + b$
$a = \sigma(z) = \frac{1}{1 + e^{-z}}$
Loss Function:
$J = -\frac{1}{m}\sum_{i=1}^{m}[y^{(i)}\log(a^{(i)}) + (1-y^{(i)})\log(1-a^{(i)})]$
Gradient Descent:
$w = w - \alpha \cdot \frac{\partial J}{\partial w}$
$b = b - \alpha \cdot \frac{\partial J}{\partial b}$

💻 Installation

Prérequis

  • Python 3.7+
  • pip (gestionnaire de paquets Python)
  • Git (pour cloner le dépôt)

Étapes d'installation

1. Cloner le dépôt
git clone https://github.com/leadylearn/logistic-regression-neural-network.git
cd logistic-regression-neural-network
2. Créer un environnement virtuel (recommandé)
python -m venv venv
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate
3. Installer les dépendances
pip install flask numpy matplotlib pillow scipy h5py scikit-image
4. Lancer l'application
python app.py
5. Accéder à l'application

Ouvrez votre navigateur et allez sur: http://localhost:5000

🔌 Référence API

Endpoints disponibles

GET /

Description: Page d'accueil avec interface de prédiction

Retour: HTML - Page principale

POST /predict

Description: Prédire si une image contient un chat

Paramètres:

  • image (file): Fichier image à classifier

💡 Exemples d'utilisation

Exemple 1: Utilisation avec curl

curl -X POST -F "image=@/path/to/image.jpg" http://localhost:5000/predict

Exemple 2: Utilisation avec Python

import requests

# URL de l'API
url = "http://localhost:5000/predict"

# Préparer le fichier
with open('image.jpg', 'rb') as f:
  files = {'image': f}
  response = requests.post(url, files=files)

# Résultat
result = response.json()
print(f"Prédiction: {result['prediction']}")
print(f"Confiance: {result['confidence']:.2%}")

Exemple 3: Utilisation avec JavaScript

const formData = new FormData();
formData.append('image', fileInput.files[0]);

fetch('/predict', {
  method: 'POST',
  body: formData
})
.then(response => response.json())
.then(data => {
  console.log('Prédiction:', data.prediction);
  console.log('Confiance:', data.confidence);
});

📊 Performance et métriques

📈 Métriques du modèle
  • Accuracy (entraînement): ~99%
  • Accuracy (test): ~70%
  • Dataset: 209 images train, 50 images test
  • Temps de prédiction: <100ms
  • Taille du modèle: ~100KB
⚡ Performance système
  • Temps de réponse API: 200-500ms
  • Memory usage: ~50MB
  • Concurrent users: ~10 (développement)
  • Image processing: 64x64 → 12288 features
  • Framework: Flask (development server)

Limitations connues

  • 🔶 Overfitting: Accuracy train >> Accuracy test
  • 🖼️ Résolution limitée: Uniquement 64x64 pixels
  • 🎯 Classification binaire: Uniquement chat/non-chat
  • 📊 Dataset petit: 259 images totales
  • 🔄 Pas de data augmentation: Variabilité limitée

🔧 Dépannage

Problèmes courants

❌ "ModuleNotFoundError: No module named 'flask'"

Cause: Flask n'est pas installé

Solution: pip install flask

❌ "Model not loaded" erreur

Cause: Les données d'entraînement ne sont pas trouvées

Solution: Vérifiez que le dossier "datasets" avec les fichiers .h5 est présent

❌ "Permission denied" sur le port 5000

Cause: Le port est déjà utilisé

Solution: Changez le port: app.run(port=5001)

❌ Images non reconnues

Cause: Format d'image non supporté ou corrompu

Solution: Utilisez des images JPG/PNG valides

Conseils d'optimisation

  • 🚀 Utiliser un serveur de production: Gunicorn + Nginx
  • 💾 Cache du modèle: Éviter de ré-entraîner à chaque démarrage
  • 🔄 Asynchronous processing: Pour les gros volumes
  • 📊 Monitoring: Ajouter des logs et métriques
  • 🔒 Sécurité: Validation des inputs et rate limiting
